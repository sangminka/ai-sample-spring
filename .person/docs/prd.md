# 요구사항 정의서: 심플 다중 사용자 블로그

## 1. 프로젝트 개요

본 프로젝트는 모든 사용자가 하나의 공용 피드에서 텍스트 기반의 게시글과 댓글을 통해 자유롭게 소통할 수 있는 심플한 블로그 서비스를 목표로 합니다.

## 2. 사용자 역할

- **비회원 (Guest)**: 로그인 없이 게시글을 조회하고 검색할 수 있는 사용자.
- **회원 (Member)**: 회원가입 후 글과 댓글을 작성하고 자신의 콘텐츠를 관리할 수 있는 사용자.

## 3. 기능 요구사항

### 3.1 회원 관리 (Authentication)

- **회원가입**: 아이디, 비밀번호 등을 입력하여 계정을 생성한다.
- **로그인**: 가입된 정보로 인증을 수행한다.
- **회원 탈퇴**: 탈퇴 시 해당 사용자가 작성한 모든 게시글과 댓글은 화면에서 제거(비노출) 처리한다.

### 3.2 게시글 기능 (Posts)

- **글 목록 조회**: 모든 사용자가 작성한 글을 최신순으로 조회한다. (페이징 적용)
- **글 상세 조회**: 특정 게시글의 전체 본문과 댓글 목록을 조회한다.
- **글 작성**: 로그인한 회원이 텍스트 전용으로 글을 작성한다. (에디터/이미지 미지원)
- **글 수정/삭제**: 본인이 작성한 글에 대해서만 수정 및 삭제 권한을 갖는다.
- **글 검색**: 제목 또는 본문에 포함된 키워드로 게시글을 검색한다.

### 3.3 댓글 기능 (Comments)

- **댓글 작성**: 로그인한 회원이 게시글에 댓글을 남긴다.
- **댓글 삭제**: 본인이 작성한 댓글에 대해서만 삭제 권한을 갖는다.
- **댓글 구조**: 대댓글 없이 게시글에 직접 달리는 1단 구조를 유지한다.

## 4. 비기능 및 제약 사항

- **권한 관리**: 관리자(Admin) 역할은 별도로 두지 않으며, 각 사용자는 자신의 리소스만 관리한다.
- **데이터 노출**: 비회원에게는 모든 게시글의 읽기 권한을 허용한다.
- **UI 단순화**: 복잡한 카테고리나 태그, 개인 블로그 페이지는 제공하지 않는다.

## 5. 수용 기준 (Acceptance Criteria)

1. 사용자가 최신순으로 정렬된 게시글 목록을 페이지별로 확인할 수 있는가?
2. 검색어 입력 시 제목과 본문에서 해당 키워드가 포함된 글이 필터링되는가?
3. 로그인한 사용자만 글/댓글을 작성할 수 있으며, 본인의 것만 관리할 수 있는가?
4. 회원 탈퇴 시 해당 사용자의 콘텐츠가 즉시 모든 목록 및 상세 뷰에서 사라지는가?
