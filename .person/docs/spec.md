# 기능 명세서

### 1. 문서 목적

본 문서는 [.person/docs/prd.md](C:\workspace\sample_lab\ai-sample-spring\.person\docs\prd.md)를 바탕으로 심플 다중 사용자 블로그 서비스의 기능을 구현 가능한 수준으로 구체화한 명세서다.

### 2. 범위

- 텍스트 기반 게시글 작성 및 조회
- 게시글 단위 댓글 작성 및 조회
- 회원가입, 로그인, 회원 탈퇴
- 게시글 검색
- 본인 리소스에 대한 수정/삭제 권한 제어

### 3. 사용자 역할

- Guest
  - 로그인 없이 게시글 목록, 상세, 검색 결과를 조회할 수 있다.
- Member
  - 회원가입 및 로그인 후 게시글과 댓글을 작성할 수 있다.
  - 본인이 작성한 게시글과 댓글만 수정 또는 삭제할 수 있다.

### 4. 도메인 정의

- 회원(User)
  - 서비스를 사용하는 계정 주체
- 게시글(Post, 구현체 명칭은 `Board`)
  - 회원이 작성한 텍스트 기반 콘텐츠
- 댓글(Comment, 구현체 명칭은 `Reply`)
  - 특정 게시글에 종속되는 1단계 댓글

### 5. 공통 정책

- 인증이 필요한 기능은 로그인 상태의 Member만 수행할 수 있다.
- Guest는 읽기 전용 사용자다.
- 관리자 역할은 없다.
- 삭제된 게시글/댓글/탈퇴 회원 콘텐츠는 화면에서 노출되지 않아야 한다.
- 인증 방식은 세션 기반으로 한다.
- 로그인 성공 시 서버 세션에 사용자 인증 정보를 저장한다.
- 로그아웃은 세션 무효화 방식으로 처리한다.
- 회원, 게시글, 댓글 삭제는 소프트 삭제를 기본으로 한다.
- 회원 탈퇴 시 해당 사용자의 게시글과 댓글은 탈퇴 시점에 즉시 일괄 비노출 처리한다.
- 소프트 삭제된 데이터는 복구 없이 영구 비활성 상태로 유지한다.
- 사용자 식별값은 로그인 아이디만 사용하며 별도 닉네임은 두지 않는다.
- 이미지, 파일 첨부, 태그, 카테고리, 대댓글 기능은 범위에서 제외한다.

### 6. 기능 상세 명세

#### 6.1 회원가입

- 목적
  - 신규 사용자가 계정을 생성한다.
- 입력
  - 아이디
  - 비밀번호
- 처리 규칙
  - 아이디는 시스템 내에서 유일해야 한다.
  - 비밀번호는 평문 저장을 허용하지 않는다.
  - 필수값 누락 시 가입은 실패해야 한다.
- 결과
  - 회원 레코드가 생성된다.
  - 회원가입 성공 시 가입 완료 응답을 반환한다.
- 예외
  - 중복 아이디
  - 유효하지 않은 입력값

#### 6.2 로그인

- 목적
  - 기존 회원이 인증을 수행한다.
- 입력
  - 아이디
  - 비밀번호
- 처리 규칙
  - 아이디와 비밀번호가 일치해야 한다.
  - 탈퇴 처리된 계정은 로그인할 수 없다.
- 결과
  - 인증 성공 시 서버 세션이 생성된다.
- 예외
  - 존재하지 않는 아이디
  - 비밀번호 불일치
  - 탈퇴 회원 로그인 시도

#### 6.2.1 로그아웃

- 목적
  - 로그인한 사용자가 현재 클라이언트에서 인증 상태를 종료한다.
- 처리 규칙
  - 로그아웃 시 서버 세션을 무효화한다.
- 결과
  - 이후 요청은 인증되지 않은 사용자로 처리된다.

#### 6.3 회원 탈퇴

- 목적
  - 회원이 자신의 계정을 탈퇴한다.
- 처리 규칙
  - 본인 인증 상태에서만 가능하다.
  - 탈퇴 후 해당 사용자가 작성한 게시글과 댓글은 모든 화면에서 비노출 처리된다.
  - 회원 삭제는 소프트 삭제로 처리하며 사용자 관점에서는 즉시 사라져야 한다.
- 결과
  - 계정은 소프트 삭제 상태가 된다.
  - 관련 콘텐츠는 목록, 상세, 검색 결과에서 조회되지 않는다.
  - 기존 세션은 더 이상 유효하지 않도록 정리되어야 한다.

#### 6.4 게시글 목록 조회

- 목적
  - 사용자가 게시글 목록을 최신순으로 조회한다.
- 사용자
  - Guest, Member
- 처리 규칙
  - 최신 작성일 기준 내림차순 정렬
  - 페이징 적용
  - 삭제된 게시글 및 탈퇴 회원의 게시글은 제외
- 출력 항목
  - 게시글 ID
  - 제목
  - 작성자명 또는 작성자 식별값
  - 작성일시
  - 댓글 수
- 예외
  - 게시글이 없으면 빈 목록 반환

#### 6.5 게시글 상세 조회

- 목적
  - 특정 게시글의 본문과 댓글을 조회한다.
- 사용자
  - Guest, Member
- 처리 규칙
  - 삭제되었거나 비노출 대상인 게시글은 조회할 수 없다.
  - 댓글은 게시글에 속한 1단계 댓글만 노출한다.
  - 삭제된 댓글 및 탈퇴 회원의 댓글은 제외한다.
- 출력 항목
  - 게시글 기본 정보
  - 게시글 본문
  - 댓글 목록

#### 6.6 게시글 작성

- 목적
  - 로그인한 회원이 게시글을 작성한다.
- 사용자
  - Member
- 입력
  - 제목
  - 본문
- 처리 규칙
  - 텍스트만 허용한다.
  - 제목과 본문은 필수값이다.
  - 작성자는 현재 로그인 사용자로 고정한다.
- 결과
  - 신규 게시글이 생성된다.

#### 6.7 게시글 수정

- 목적
  - 작성자가 자신의 게시글을 수정한다.
- 사용자
  - Member
- 입력
  - 제목
  - 본문
- 처리 규칙
  - 작성자 본인만 수정할 수 있다.
  - 삭제된 게시글은 수정할 수 없다.
- 결과
  - 게시글 내용이 갱신된다.
- 예외
  - 권한 없음
  - 존재하지 않는 게시글

#### 6.8 게시글 삭제

- 목적
  - 작성자가 자신의 게시글을 삭제한다.
- 사용자
  - Member
- 처리 규칙
  - 작성자 본인만 삭제할 수 있다.
  - 삭제 후 목록, 상세, 검색 결과에서 즉시 비노출되어야 한다.
  - 삭제된 게시글은 화면에 자리표시 없이 완전히 숨긴다.
- 결과
  - 게시글은 소프트 삭제 상태로 변경된다.

#### 6.9 게시글 검색

- 목적
  - 사용자가 키워드로 게시글을 검색한다.
- 사용자
  - Guest, Member
- 입력
  - 검색어
- 처리 규칙
  - 제목 또는 본문에 검색어가 포함된 게시글을 반환한다.
  - 결과 정렬은 최신순을 기본으로 한다.
  - 삭제된 게시글 및 탈퇴 회원 콘텐츠는 제외한다.
  - 빈 검색어 입력 시 전체 목록 조회와 동일하게 처리한다.
- 출력
  - 게시글 목록 조회와 동일한 수준의 목록 데이터

#### 6.10 댓글 작성

- 목적
  - 로그인한 회원이 게시글에 댓글을 작성한다.
- 사용자
  - Member
- 입력
  - 댓글 본문
  - 대상 게시글 ID
- 처리 규칙
  - 댓글은 게시글에 직접 연결되는 1단 구조만 허용한다.
  - 삭제되었거나 존재하지 않는 게시글에는 댓글을 작성할 수 없다.
  - 본문은 필수값이다.
- 결과
  - 댓글이 생성되고 해당 게시글 상세에 노출된다.

#### 6.11 댓글 삭제

- 목적
  - 작성자가 자신의 댓글을 삭제한다.
- 사용자
  - Member
- 처리 규칙
  - 작성자 본인만 삭제할 수 있다.
  - 삭제 후 게시글 상세에서 즉시 비노출되어야 한다.
  - 삭제된 댓글은 화면에 자리표시 없이 완전히 숨긴다.
- 예외
  - 권한 없음
  - 존재하지 않는 댓글

### 7. 권한 매트릭스

| 기능             | Guest | Member      |
| ---------------- | ----- | ----------- |
| 회원가입         | 가능  | 가능        |
| 로그인           | 가능  | 가능        |
| 게시글 목록 조회 | 가능  | 가능        |
| 게시글 상세 조회 | 가능  | 가능        |
| 게시글 검색      | 가능  | 가능        |
| 게시글 작성      | 불가  | 가능        |
| 게시글 수정      | 불가  | 본인만 가능 |
| 게시글 삭제      | 불가  | 본인만 가능 |
| 댓글 작성        | 불가  | 가능        |
| 댓글 삭제        | 불가  | 본인만 가능 |
| 회원 탈퇴        | 불가  | 본인만 가능 |

### 8. 화면/응답 상태 규칙

- 인증이 필요한 기능에 Guest가 접근하면 로그인 필요 응답을 반환해야 한다.
- 본인 소유 리소스가 아닌 경우 권한 없음 응답을 반환해야 한다.
- 존재하지 않거나 비노출 대상인 게시글/댓글은 조회 불가 응답을 반환해야 한다.
- 목록 조회 결과가 없더라도 시스템 오류로 처리하지 않고 빈 결과를 정상 반환해야 한다.

### 9. 비기능 요구사항

- 게시글 목록과 검색 결과는 페이징 가능해야 한다.
- 게시글 상세 진입 시 댓글 목록을 함께 조회할 수 있어야 한다.
- 탈퇴 및 삭제 반영은 사용자 화면 기준 즉시 일관되게 적용되어야 한다.
- 로그인은 세션 기반 인증을 사용해야 한다.
- 인증된 사용자 정보는 서버 세션으로 식별 가능해야 한다.
- 로그아웃 시 세션이 무효화되어야 한다.
- 화면 표시용 사용자명은 로그인 아이디와 동일해야 한다.
- 용어는 구현체 패키지 기준으로 `User`, `Board`, `Reply`를 사용해도 무방하나, 사용자 문서상 의미는 회원/게시글/댓글으로 동일해야 한다.

### 11. 수용 기준

- [ ] Guest는 로그인 없이 게시글 목록과 상세를 조회할 수 있다.
- [ ] 게시글 목록은 최신순이며 페이징이 적용된다.
- [ ] 검색어 입력 시 제목 또는 본문에 검색어가 포함된 게시글만 조회된다.
- [ ] Member만 게시글과 댓글을 작성할 수 있다.
- [ ] 게시글 수정/삭제는 작성자 본인만 가능하다.
- [ ] 댓글 삭제는 작성자 본인만 가능하다.
- [ ] 회원 탈퇴 시 해당 사용자의 게시글과 댓글은 모든 화면에서 즉시 비노출된다.
- [ ] 삭제되거나 비노출 대상인 게시글/댓글은 조회 결과에 포함되지 않는다.
