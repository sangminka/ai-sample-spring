# Step4 페이지번호 UI 구조

- 날짜: 2026-03-23
- 참여: 사용자 + Codex

## 배경

`todo.md`의 Step 4를 진행하기 위해 전체 게시글 수 계산과 페이지 번호 UI 구조를 논의했다.

Step 4의 핵심은 다음과 같다.

- `COUNT(*)`로 전체 게시글 수를 구한다.
- 전체 개수와 페이지 크기(3)를 기준으로 총 페이지 수를 계산한다.
- Mustache에서 `1 2 3 4 ...` 형태의 페이지 번호 목록을 동적으로 출력한다.

이 과정에서 페이지 번호 목록을 `List<Integer>`로 둘지,
Mustache에 맞춘 객체 리스트로 둘지 결정할 필요가 있었다.

## 핵심 논의

- Step 4는 학습용 구현으로 진행하는 것이 좋다.
- 따라서 페이지 번호는 축약형(`1 ... 8 9 10 ... 100`)이 아니라 전체 번호를 전부 출력하는 방식으로 간다.
- 다만 Mustache는 반복 중인 숫자와 `currentPage`를 비교하는 표현이 어렵다.
- 따라서 현재 페이지를 번호 목록 안에서 다르게 렌더링하려면 객체 리스트 구조가 더 적합하다.
- `currentPage`만 있으면 현재 페이지 숫자를 따로 보여줄 수는 있지만,
  번호 목록 안에서 현재 번호만 강조하는 처리는 어렵다.

## 결론

- Step 4는 학습용 단순 구현으로 진행한다.
- 페이지 번호 UI는 전체 페이지 번호를 모두 출력하는 방식으로 간다.
- `pageNumbers`는 `List<Integer>`가 아니라 객체 리스트로 설계한다.
- 객체 리스트의 각 원소는 최소한 다음 값을 가진다.
  - `number`
  - `current`
- Mustache는 `{{#current}}`, `{{^current}}` 블록으로 현재 페이지를 다르게 렌더링한다.

## 권장 구조

```java
public class BoardResponse {

    @Data
    public static class PageNumberResponse {
        private Integer number;
        private boolean current;

        public PageNumberResponse(Integer number, Integer currentPage) {
            this.number = number;
            this.current = number.equals(currentPage);
        }
    }

    @Data
    public static class PagingResponse {
        private Integer currentPage;
        private Integer pageSize;
        private Integer totalCount;
        private Integer totalPageCount;
        private boolean hasPrev;
        private Integer prevPage;
        private boolean hasNext;
        private Integer nextPage;
        private List<PageNumberResponse> pageNumbers;
    }
}
```

## Mustache 예시

```mustache
{{#model.paging.pageNumbers}}
    {{#current}}
        <span class="page-number page-number--active">{{number}}</span>
    {{/current}}

    {{^current}}
        <a href="/board/list?page={{number}}" class="page-number">{{number}}</a>
    {{/current}}
{{/model.paging.pageNumbers}}
```
